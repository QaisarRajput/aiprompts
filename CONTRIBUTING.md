# Contributing

## Development

- Use Node.js 20+ and pnpm.
- Run `pnpm install` once, then use root scripts (`pnpm lint`, `pnpm typecheck`, `pnpm test`).

## Data policy

- `data/` is generated by the ETL and committed to git.
- Do not hand-edit files under `data/`.
- Use ETL inputs (including `overrides.json` once added) to change generated output.

## Commit style

- Prefer Conventional Commits, e.g. `feat:`, `fix:`, `chore:`.
